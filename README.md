# 🍔 Swiggy — PM Product Teardown Case Study

> **A structured product management analysis of 
> Swiggy — identifying core user pain points and 
> proposing data-driven feature solutions using 
> RICE prioritization and North Star metrics.**

---

## 👤 About This Project

| Field | Details |
|-------|---------|
| Role | Product Manager (Simulated) |
| Product Analyzed | Swiggy — India's leading food delivery app |
| Date | April 2026 |
| Tools Used | Google Docs, Google Sheets |
| Frameworks | RICE, North Star Metrics, HEART, Root Cause Analysis |

This project simulates a PM product teardown — 
a structured analysis used by real PMs to deeply 
understand a product, identify gaps, and propose 
improvements. I analyzed Swiggy across 3 user 
personas, identified 5 critical pain points, 
proposed 5 prioritized feature solutions, and 
defined success metrics for each.

---

## 🎯 Problem Statement

Swiggy has 100+ million users but faces significant 
user experience gaps — especially around delivery 
time accuracy, group ordering, and coupon failures. 
These pain points directly impact user retention 
and trust. This teardown identifies the root causes 
and proposes high-impact, low-effort solutions 
prioritized using the RICE framework.

## 🔢 RICE Prioritization Results

| Feature | Pain Point | Reach | Impact | Confidence | Effort | RICE Score | Priority |
|---------|-----------|-------|--------|------------|--------|------------|----------|
| Auto Refund System | Missing items | 6 | 3 | 85% | 2 | **7.7** | P1 |
| Smart ETA Engine | Inaccurate ETA | 9 | 3 | 80% | 3 | **7.2** | P1 |
| Smart Coupon Validator | Coupon failures | 8 | 2 | 90% | 2 | **7.2** | P1 |
| Group Order Feature | No group ordering | 7 | 3 | 70% | 5 | **2.9** | P2 |
| Scheduled Delivery | No scheduling | 5 | 2 | 60% | 4 | **1.5** | P3 |
